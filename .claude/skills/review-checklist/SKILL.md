# PR レビューチェックリスト — MiniPOSKids

PR をレビューする際の確認観点リストです。
レビュー依頼を受けたとき・自分でセルフレビューするときに参照してください。

## 必須チェック項目

### コード品質
- [ ] Swift コーディング規約（`swift-conventions`）に沿っているか
- [ ] 変数・関数名が意図を正確に表しているか
- [ ] マジックナンバーが定数化されているか
- [ ] body が大きすぎる View はサブViewに分割されているか

### バグ・安全性
- [ ] オプショナルの強制アンラップ（`!`）が不必要に使われていないか
- [ ] メインスレッドで UI 更新を行っているか
- [ ] 非同期処理でメモリリーク（強参照サイクル）がないか
- [ ] エラーハンドリングが適切か

### セキュリティ
- [ ] APIキー・パスワードがコードにハードコードされていないか
- [ ] ユーザー入力のバリデーションが行われているか
- [ ] `UserDefaults` に機密情報を保存していないか（Keychain を使用）

### SwiftUI 固有
- [ ] `@State` に `private` がついているか
- [ ] `NavigationView`（非推奨）でなく `NavigationStack` を使っているか
- [ ] `#Preview` が正常に動作するか

### テスト
- [ ] 新機能・バグ修正に対応するテストが追加されているか
- [ ] 既存テストが通過しているか

## 推奨チェック項目

- [ ] PR の変更量が適切か（大きすぎる場合は分割を提案）
- [ ] コミットメッセージが規約（`git-workflow`）に沿っているか
- [ ] 不要なコメントアウトや `print()` デバッグが残っていないか
- [ ] TODO コメントがある場合、チケット・Issue と紐づいているか

## レビューコメントの書き方

- **必須修正**: `[must]` プレフィックスをつける
- **推奨**: `[nit]` プレフィックスをつける
- **質問**: `[ask]` プレフィックスをつける
- 指摘するだけでなく、改善案も添える
